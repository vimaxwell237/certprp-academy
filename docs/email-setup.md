# Email Setup Guide

This project uses two separate email paths:

1. `info@certprep.it.com` as a real inbox for inbound mail from learners.
2. Transactional email for account confirmation, password reset, reminders, and notifications.

Those are related, but they are not the same system.

## What Supabase can and cannot do

Supabase Auth can send account emails such as:

- sign-up confirmations
- password reset emails
- email change confirmations
- magic link or OTP emails

Supabase does **not** host a mailbox inbox for `info@certprep.it.com`.

If you want to **receive** mail at `info@certprep.it.com`, you need an email hosting provider for your domain and working MX records. That mailbox provider is separate from your Supabase project.

## Recommended setup

You can use `info@certprep.it.com` for both:

- inbound support and general messages
- Supabase Auth verification and password reset emails
- app-side reminders and notification emails

That is a valid starting setup for this app.

Later, if deliverability or mailbox noise becomes a problem, move automated mail to
`no-reply@certprep.it.com` or `auth@certprep.it.com`.

## Part 1: Set up the real inbox

In your DNS and mail host:

1. Create the mailbox `info@certprep.it.com`.
2. Point the domain MX records to your mailbox provider.
3. Publish the SPF, DKIM, and DMARC records required by that provider.
4. Confirm you can send to and receive mail from `info@certprep.it.com`.

This is what enables inbound mail reception. Supabase is not involved in this part.

## Part 2: Configure Supabase Auth email sending

In the Supabase dashboard:

1. Open `Authentication`.
2. Open the email provider settings.
3. Configure a custom SMTP provider using your mail provider's SMTP credentials.
4. Set the sender email to `info@certprep.it.com`.
5. Set the sender name to `CertPrep Academy`.
6. Keep email confirmations enabled.
7. In the URL configuration, set:
   - `Site URL` to `https://www.certprep.it.com`
   - `Redirect URLs` to include:
     - `http://localhost:3000/auth/callback`
     - `https://www.certprep.it.com/auth/callback`
     - `https://certprep.it.com/auth/callback` if you also serve the root domain
8. Customize the confirmation and reset templates if you want the visible sender copy to match the brand.

This app already sends:

- sign-up users through `/auth/callback?next=/login&mode=confirm`
- password reset users through `/auth/callback?next=/reset-password&mode=recovery`

So the callback route above must be present in Supabase Redirect URLs.

## Part 3: Configure app notification emails

The app's own notification delivery pipeline now supports either:

- generic SMTP, or
- Resend's API

SMTP takes priority when both are configured.

### Option A: SMTP

Set these environment variables:

```env
APP_BASE_URL=https://www.certprep.it.com
NEXT_PUBLIC_APP_URL=https://www.certprep.it.com
AUTOMATION_SECRET=replace-with-a-long-random-secret

NOTIFICATION_FROM_EMAIL=info@certprep.it.com
EMAIL_FROM_ADDRESS=info@certprep.it.com
EMAIL_SMTP_HOST=your-smtp-host
EMAIL_SMTP_PORT=587
EMAIL_SMTP_USERNAME=your-smtp-username
EMAIL_SMTP_PASSWORD=your-smtp-password
EMAIL_SMTP_SECURE=false
```

Use `EMAIL_SMTP_SECURE=true` when your SMTP provider requires implicit TLS, which is commonly paired with port `465`.

### Option B: Resend API

Set these environment variables instead:

```env
APP_BASE_URL=https://www.certprep.it.com
NEXT_PUBLIC_APP_URL=https://www.certprep.it.com
AUTOMATION_SECRET=replace-with-a-long-random-secret

NOTIFICATION_FROM_EMAIL=info@certprep.it.com
EMAIL_FROM_ADDRESS=info@certprep.it.com
RESEND_API_KEY=re_xxxxxxxxx
```

## Part 4: Schedule notification processing

Transactional emails generated by the app are sent through the internal automation routes.

Set these scheduler jobs against production:

- `POST /api/internal/process-scheduled-jobs`
- `POST /api/internal/process-notification-deliveries`

Use either of these headers:

- `Authorization: Bearer <AUTOMATION_SECRET>`
- `x-automation-secret: <AUTOMATION_SECRET>`

Suggested baseline frequency:

- every 5 minutes for both routes

## Production checklist

Before going live, confirm:

1. `info@certprep.it.com` can receive inbound mail.
2. Supabase sign-up confirmation emails are delivered to a real external inbox.
3. Password reset emails arrive correctly.
4. The app can send notification emails from `/api/internal/process-notification-deliveries`.
5. Links in emails point to `https://www.certprep.it.com`.
6. SPF, DKIM, and DMARC all validate for your sending domain.
7. Your provider-specific sending limits are acceptable for the traffic you expect.

## Important note on "unlimited"

No serious email system is truly unlimited.

What this setup gives you is:

- no dependency on Supabase's starter email restrictions
- production-grade auth email sending through your own SMTP provider
- a real hosted inbox for `info@certprep.it.com`
- app-side notification delivery through the same provider or through Resend

Your real throughput and mailbox limits will come from the email provider plan you choose.
